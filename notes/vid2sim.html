Vid2Sim: 从 monocular video 到可交互 urban simulator

论文：Vid2Sim: Realistic and Interactive Simulation from Video for Urban Navigation
作者：Ziyang Xie, Zhizheng Liu, Zhenghao Peng, Wayne Wu, Bolei Zhou
版本：arXiv:2501.06693v2, 2025-01-14；本地 PDF：ref/Vid2Sim_ Realistic and Interactive Simulation from Video for Urban Navigation.pdf
阅读目标：面向 collaborative real2sim simulation platform，重点还原基础 Vid2Sim pipeline，而不是把它强行解释为 world model。

这篇论文在 real2sim 里的位置

一句话：Vid2Sim 把一段手持单目街景视频转换成一个可用于 closed-loop RL training 的 digital twin simulator。它的底层逻辑是“Gaussian Splatting 负责真实视觉，mesh/Unity physics 负责碰撞和交互”，再用静态/动态障碍和天气、光照、季节编辑扩充训练分布。

它不是以“预测未观测世界状态”为核心的 world model。论文里的 unseen/partial 问题主要通过两类工程手段缓解：第一，用 monocular depth/normal priors 让 GS 更像几何表面；第二，把训练相机轨迹之外的渲染 artifact 用 screen-space covariance culling 剪掉。真正的 3D completion、object generation、material/physics inference 并不是本文主菜。

对我们的平台更重要的是：它给出了一个可落地的 real2sim 最小闭环：视频采集/下载、重建、mesh baking、simulator integration、agent training、sim2real evaluation。这是“基础 Vid2Sim 方法”的价值。

控制流版本

  1. 从 web/实拍视频取 15s clip，做人脸和隐私信息遮挡。
  2. 用 DEVA 生成 moving object mask，避免动态行人/车辆污染静态场景重建。
  3. 用 GLOMAP 做 Structure-from-Motion，得到 camera poses 和初始化 point cloud；论文认为 in-the-wild 视频下 COLMAP 更容易失败。
  4. 训练 Gaussian Splatting，在常规 RGB photometric loss 外加入 depth/normal/geometry/scale regularization。
  5. 从训练好的 GS 渲染多帧 depth，融合成 TSDF，再抽取 mesh；ground plane 单独处理，非地面 mesh 用于碰撞。
  6. 在 Unity 中同时导入 GS 和 mesh：GS 用 custom shader 渲染，mesh 设为 invisible collider。
  7. 把 static obstacles、dynamic pedestrians、weather/lighting/season variants 组合进场景，形成多样化训练分布。
  8. 用 Gym wrapper 接入 RL 框架，训练视觉导航 agent；最后直接在真实机器人上 zero-shot test。

重建部分：让 GS 不只是好看，还能导出可碰撞几何

标准 3DGS 的强项是 novel view rendering，不是几何正确性。Vid2Sim 的第一个核心贡献，就是把 monocular geometry cues 塞进 GS 训练，让渲染出来的 depth/normal 更适合后续 mesh extraction。

Geometry supervision 的来源

  • Depth Anything V2 之类的 monocular depth estimator 提供 depth prior，但这类 depth 通常是 relative depth，不是绝对 metric depth。
  • pseudo normal 不是人工标注，而是从 predicted depth 反投影成 point cloud 后用 PCA 估计局部法线。
  • GS 自己也能 render depth 和 normal：depth 来自 splat 到相机的距离，normal 来自 covariance shortest axis 的方向。

Loss/objective 清单

Objective 作用 为什么这样设计
L_rgb 常规 2D photometric rendering loss。 保证 novel view RGB 质量，是 GS 的基础目标。
L_depth patch-based NCC between rendered depth and predicted depth。 避免直接匹配相对 depth 的绝对数值；关注局部结构相似性，对 scale shift、noise、occlusion 更稳。
L_normal rendered normal 与 pseudo normal 的 cosine distance。 让表面方向更一致，补足 depth 只约束距离、不直接约束表面朝向的问题。
L_geo 相邻 pixel 的 rendered normals 在低 depth-gradient 区域保持一致。 平坦区域应当法线平滑；用 depth gradient 做权重，避免跨边界强行平滑。
L_scale 最小化 Gaussian covariance 的 shortest axis scale。 借鉴 2DGS，鼓励 splats 更像 surface disks，利于几何表面表达。

最终训练目标是这些项的和：L_total = L_rgb + L_depth + L_normal + L_geo + L_scale。Supplementary 给出实现细节：训练 30000 iterations，采用 3DGS hyper-parameters 和 AbsGS densification strategy；depth/normal/geometry-consistency losses 从第 500 iteration 开始加入。

Screen-space covariance culling

RL agent 在模拟器里会走到训练视频相机从未到过的位置，尤其低机位、近地面视角会出现 floaters。Vid2Sim 的处理是：把每个 Gaussian 投影到 2D screen 后，如果 2D covariance 过大，就把它 cull 掉。论文的判据是 ||Sigma'||_inf > alpha * A_img。

这不是重训，而是 post-training rendering fix。它的工程价值很大：real2sim 平台只要允许 agent 自由走，就一定会遇到 train-view distribution 外的 observation artifact。Supplementary 的相对评估显示，使用 culling 后 rendered observations 与训练视角分布的 FID 从 214.47 降到 191.54，改善约 10.7%。

仿真部分：视觉和物理分开，但在同一坐标系里闭合

视觉层：GS

负责给 agent 提供 photorealistic RGB observation，也可以渲染 depth sensor。Unity 中通过 custom shader 实时渲染 Gaussian Splats。

物理层：mesh collider

从 GS depth 融合得到 TSDF，再提取 mesh。mesh material 设为 invisible，只做 collision 和 interaction primitive。

Mesh extraction 和 ground handling

论文从每个训练视角渲染 depth map，用 KinectFusion 风格的 TSDF 融合，voxel size 设为 0.1。为了避免不平整 ground mesh 影响机器人动力学，作者会移除 ground plane：先用 SAM-HQ 得到 ground segmentation，再用 rendered normal 的均值方向做 normal prior，保留与地面平均法线夹角小于 15 度的像素作为更干净的 ground mask。

在 Unity 里，ground surface 被配置成水平可行走区域；其他 scene mesh 和障碍物被 tag 成 collidable objects。这个选择很实用：视觉上仍看到真实街景地面，物理上则避免坑洼重建导致机器人抖动或误碰撞。

Interactive scene composition

  • Static obstacles：traffic cones、trash bins、traffic lights、poles 等 3D assets 随机放进场景。
  • Dynamic agents：pedestrians 用 A* 在 movable area 里从随机点移动到随机点。
  • Occlusion handling：通过 GS rasterization 和 mesh rendering 的 z-buffering 组合 RGB/depth，保证前景 asset 和背景 GS 的遮挡关系正确。
  • Safety-critical scenarios：可以构造 falling trash cans、road construction、workers 等 corner cases。

Diverse scene augmentation

Vid2Sim 用 scene-level editing 扩增 lighting、season、semantics，并在 Unity 里用 particle systems 模拟 rain、fog、snow。这个部分更像训练分布扩充，不是几何/物理重建本身。对平台而言，它适合作为 scenario authoring layer：在同一个 reconstructed digital twin 上生成多种 appearance 和 traffic/obstacle variants。

RL 和 sim2real：论文如何证明模拟器真的有用

Agent observation/action

  • 机器人：四轮 delivery robot，前置 RGB camera。
  • Observation：当前 RGB 加过去 5 个 timestep 的 image stack，再拼接 goal distance 和 heading angle。真实部署中相机 1280x720，resize 到 128x72。
  • Action：两个连续控制量，范围 [-1, 1]；论文主文描述为轮速/速度调制，supplementary 描述为 normalized linear/angular velocities。真实部署时通过 system identification 把归一化速度重标定到真实机器人单位。
  • 训练接口：Unity environment compiled 后通过 wrapper 兼容 OpenAI Gym 和 Stable-Baselines3；simulator 50Hz，RL 5Hz。

Reward shaping

Reward term 论文设置 平台移植提醒
R_term 到达目标 +10，失败 -10。 直接可用；失败条件要和平台安全规则绑定。
R_dist 按 goal distance 变化给 dense reward，权重 c1 = 1。 论文公式写作 d_t - d_{t-1}，但文字说鼓励靠近目标；实现时要核对符号。
R_steer -||s_t - s_{t-1}|| * v_t，权重 c2 = 0.05。 用于减少不稳定转向；适合真实机器人。
R_crash 碰撞给 dense negative reward，权重 c3 = 1.0。 需要稳定的 collision event；这依赖 mesh collider 质量。
R_time 鼓励更短 episode；论文文字给出 R_time = -Delta t 和 c4 = -0.1。 这里也有潜在符号歧义；平台复现前应检查作者代码或自行设定。

Evaluation metrics

Rendering: PSNRRendering: SSIMRendering: LPIPSArtifact: FIDNavigation: SRNavigation: SPLSafety: Cost / hit numberSocialNav: SNS

这套指标组合很适合我们的平台：rendering metrics 只说明“看起来像不像”，navigation metrics 才说明“训练出来的策略能不能用”。Vid2Sim 的评估逻辑不是单纯拼视觉质量，而是把 simulator 当作 agent training infrastructure 来验证。

结果读法：哪些数字值得记

问题 关键结果 对平台的含义
重建质量是否比 baseline 好？ Vid2Sim 达到 PSNR 32.41 / SSIM 0.927 / LPIPS 0.127，优于 3DGS、2DGS、Video2Game。 几何监督没有牺牲视觉质量，反而略有提升；说明 depth/normal priors 可作为平台默认 reconstruction regularizer。
Hybrid simulator 是否比 mesh-only 好？ PointNav 中 mesh RGB baseline SR 48.8%，Vid2Sim static/dynamic obstacles 达到 80.8%/81.6%；SocialNav 中 mesh baseline 43.2%，Vid2Sim dynamic 74.4%。 只用 textured mesh 做视觉会有明显 sim2real gap；GS visual + mesh physics 是更合理的底座。
真实世界 zero-shot 是否成立？ 真实实验里 baseline 30 envs 三个任务都是 0%；Vid2Sim 30 envs 达到 Go Straight 85%、Static Obstacle 65%、Dynamic Obstacle 55%。 数据规模直接影响泛化；30 个环境仍然偏小，但趋势说明“更多 reconstructed scenes”可能比过度调参更重要。
动态/静态障碍是否重要？ No Obj、Static、Dynamic 逐步提升，且 Static 训练在 SocialNav 上出现一定 emergent avoidance。 平台应把 object/agent insertion 当作核心数据增广，而不是后期装饰。

论文的隐含假设和 failure modes

依赖条件

  • 视频要有足够纹理、视角变化和静态背景，才能做可靠 SfM。
  • 绝对 scale 主要依赖 SfM、机器人尺寸和场景标定；monocular depth 本身不提供 metric scale。
  • 场景适合被简化为水平 walkable ground 加非地面 colliders。
  • 移动物体能被 tracker/mask 分离，否则会在 GS 和 mesh 里留下 ghost。
  • agent 的任务是地面 urban navigation，不是任意交互操作。

容易失效的地方

  • GLOMAP/SfM 失败会让后续全部漂移。
  • GS 在 train-view 之外产生 floaters，culling 只能缓解，不能保证补全真实几何。
  • mesh collider 与 GS 视觉可能不对齐，导致“看着没撞但物理撞了”或相反。
  • ground segmentation/normal prior 错误会改变可行走区域。
  • 插入的静态和动态 assets 不一定遵守真实世界语义分布。
  • 物体 material、mass、friction 等 physical parameters 没有从视频估计。
  • 论文承认环境构建耗时，主要瓶颈在 GLOMAP 初始化和 GS conversion；当前只有 30 个环境。

特别注意：Vid2Sim 的“未见区域处理”不是 generative completion。它通过 geometry regularization、view artifact culling 和场景增广提高可用性，但如果视频根本没有覆盖某条支路、建筑背面或遮挡后空间，论文并没有真正恢复它们的几何和语义。

对 collaborative real2sim 平台的迁移判断

如果把 Vid2Sim 当作平台底座，我建议按“三层架构”吸收：reconstruction layer 产出 GS + mesh + metadata，simulation layer 负责 Unity/Isaac/自研引擎里的 physics、assets 和 sensors，evaluation layer 用 agent task metrics 检查 real2sim 是否真的有用。

Transfer matrix

模块 Vid2Sim 做法 可迁移价值 移植到我们平台的建议 风险/缺口 优先级
Scene representation Gaussian Splats 做 photorealistic rendering，mesh 做 invisible collider。 高：直接解决“视觉真实”和“物理交互”难以同时满足的问题。 把 GS、collision mesh、camera calibration、scale metadata 打包成统一 scene package。 视觉-物理错位需要 QA 工具；mesh 粗糙会影响碰撞。 P0

Object completion 主要是插入 static assets 和 dynamic pedestrians，不是真正从遮挡视频里补全物体。 中：可用于 scenario densification，但不能解决 unseen geometry。 短期接 asset library 和 placement rules；长期接 3D generative completion 或 retrieval-based asset matching。 语义位置不合理会污染 policy；需要 scene graph constraints。 P1
Asset generation 论文使用常见城市物体 assets；未提出自动 asset generation。 中：作为 baseline 足够，但不覆盖开放世界长尾物体。 先做 curated asset catalog；再接 text/image-to-3D 或 scan/retrieval pipeline。 资产风格、尺度、材质、碰撞盒需要统一标准。 P2
Physical parameter estimation 主要做 robot dynamics system identification；环境物体物理参数基本没有估计。 低到中：可借鉴系统辨识思路，但论文没有完整答案。 平台需要独立模块估计或配置 friction、mass、restitution、agent dynamics。 没有真实参数时，碰撞行为可能只是“能撞”，不是“撞得对”。 P1
Simulator integration Unity custom GS shader + invisible mesh collider + Gym wrapper。 高：工程闭环清楚。 抽象成 engine adapters：Unity/Isaac/Omniverse/自研 renderer 都实现同一 scene interface。 不同引擎的 depth、z-buffer、physics step 和 sensor latency 要对齐。 P0
Dynamic scene authoring 静态障碍随机采样；pedestrian 用 A* 在可行走区域移动。 高：直接提升 navigation policy robustness。 把 object placement、traffic flow、pedestrian intent 做成可配置 scenario graph。 随机放置可能制造不现实场景；需要地图语义和可行走区域约束。 P1
Appearance/weather augmentation scene editing 改 lighting/season/semantics；Unity particle systems 模拟 rain/fog/snow。 中：能扩大视觉域，但对几何和 physics 帮助有限。 作为 training augmentation layer；评估是否真的提升 held-out/real-world SR。 编辑可能破坏 multi-view consistency；天气粒子需要与 sensor model 对齐。 P2
View artifact handling screen-space covariance culling 删除 2D 投影过大的 splats。 高：closed-loop agent 必然产生 train-view 外视角。 加入 runtime renderer filter，并记录 culling rate 作为 scene health metric。 阈值过强会删除真实大面积结构；阈值过弱不能去 floater。 P0
Evaluation metrics 重建用 PSNR/SSIM/LPIPS/FID，导航用 SR/SPL/Cost/SNS，真实部署用 task success。 高：覆盖视觉质量、交互安全和 sim2real。 平台 dashboard 同时显示 rendering metrics 和 task metrics，不用单一视觉分数判断 real2sim 质量。 没有真实部署时，需要设计 proxy tasks 和 held-out digital twins。 P0
Unseen/partial scene completion 论文基本不解决；只通过 scene augmentation 和 artifact culling 提升训练可用性。 低：不能直接支撑开放世界补全。 在 Vid2Sim 底座之外增加 semantic scene completion、layout prior、retrieval/generative 3D modules。 补全内容必须同时满足 visual plausibility、collision correctness 和 task usefulness。 P1

我会怎样把它落到我们平台

  1. 先复现 hybrid scene package：每个场景输出 scene.gs、collider_mesh.obj、camera_poses.json、scale.json、walkable_mask 和 QA report。
  2. 把 geometry regularization 做成默认训练配置：depth/normal/geo/scale losses 可开关，记录每个 loss 的曲线和 mesh quality。
  3. 给 closed-loop renderer 加 health metrics：包括 floater culling rate、agent-view FID proxy、near-ground artifact count、visual-physics alignment error。
  4. 场景编辑不要只做外观：同时维护 object placement constraints、collision proxy、semantic labels 和 dynamic agent routes。
  5. 补全模块独立设计：Vid2Sim 负责把“已观测视频”变成 simulator；unseen/partial completion 应作为上层模块，把补全结果写回 scene graph、asset graph 和 collider graph。

读完这篇应记住的核心逻辑

Vid2Sim 的本质不是“生成一个世界”，而是“把真实视频转换成一个 agent 可以闭环训练的近真实数字孪生”。它的关键设计是把 representation 拆开：GS 承担感知真实感，mesh 承担物理交互；再用 depth/normal priors 把二者尽量对齐。

对我们的 collaborative real2sim 平台来说，最该移植的是 hybrid representation、geometry-consistent GS training、screen-space artifact handling、simulator adapter 和 task-based evaluation。最不该误读的是：它还没有解决开放世界补全，也没有自动估计完整物理参数。
