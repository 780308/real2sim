ReaDy-Go: 动态 3D Gaussian Splatting real-to-sim 数据工厂

论文: ReaDy-Go: Real-to-Sim Dynamic 3D Gaussian Splatting Simulation for Environment-Specific Visual Navigation with Moving Obstacles
作者: Seungyeon Yoo, Youngseok Jang, Dabin Kim, Youngsoo Han, Seungwoo Jung, H. Jin Kim。arXiv:2602.11575v2, 2026-02-14。
本地 PDF: ref/ReaDy-Go.pdf。阅读视角: collaborative real2sim simulation platform, unseen / partially observed scenes, open-world / world-model-style completion。

先把摘要翻成我们关心的问题

ReaDy-Go 的主张是: 只用目标环境的一段静态 monocular video，重建 static GS scene，再插入可动画化的 human GS avatars，生成带 moving obstacles 的 photorealistic navigation datasets，用 imitation learning 训练 environment-specific RGB-only visual navigation policy，从而缩小 sim-to-real gap。

它不是一个通用 scene completion 或 open-world world model 论文。更准确地说，它是一个 real-to-sim 数据生成 pipeline: 让真实环境变成可渲染、可规划、可批量采样的动态仿真场。对我们的平台最有价值的是中间层接口: GS scene、voxel/occupancy map、human GS actor、2D trajectory、expert action、RGB observation。

1. 环境采集与尺度恢复

输入处理输出关键假设
目标部署环境的一段 static monocular video 用 COLMAP 估计 camera poses；初始帧中放 ArUco marker 得到 metric scale。 带 metric camera poses 的图像序列。 采集时场景基本静态；视频覆盖导航区域；尺度标定可靠。

2. Static GS Scene Reconstruction

背景环境用 PGSR 重建。GS primitive 的状态包括 position、rotation、scale、opacity、color。PGSR 相比 vanilla GS 更强调 surface reconstruction: 它把 3D Gaussians 压成近似平面，并在 photometric loss 之外使用 geometric regularization，目的是减少漂浮噪声、提高 multi-view consistency。

输入中间表示损失 / 目标输出
posed RGB frames 3D Gaussians: mean, covariance/scale/rotation, opacity, color photometric loss + geometric regularization 可 novel-view rendering、可 voxelization 的 static GS scene

3. Human GS Avatar 与 Motion Generation

动态障碍物不是 Unity mesh，而是可动画化的 human GS。作者从 NeuMan 数据集中用 HUGS 提取 human GS avatars。HUGS 将 dynamic human 和 static scene 解耦，在 canonical space 中用 SMPL 初始化，并结合 triplane features；fully connected layers 估计 human Gaussian attributes 和 LBS weights，使 human GS 能在 novel SMPL poses 下动画化。

对运动生成，输入是 human planner 给出的 2D trajectory。系统把它转换成 body root linear / rotation velocities，归一化到 HumanML3D 表示，送入 PriorMDM。PriorMDM 输出 3D body joint positions，再通过 SMPLify 拟合成 SMPL parameters，最后变换到目标环境 world coordinate。

输入模型输出我们要注意的地方
desired 2D human trajectory PriorMDM + SMPLify + HUGS human GS 逐帧 human root pose、joint configuration、可渲染 human GS 这解决的是“视觉上真实的动态人”，不是物理接触、碰撞响应或真实人群行为。

4. 从 GS 到 Planning Maps

为了给 robot planner 和 human planner 使用，ReaDy-Go 将 static GS scene 转成 occupancy maps。它先做 voxelization: 若一个 voxel 与某个 Gaussian 的 1 sigma ellipsoid 相交，则记为 occupied。由于 GS 重建在地面附近会有 spurious Gaussians，作者用 voxel 内 accumulated opacity 做过滤: opacity sum 低于阈值就当作 free space。

步骤表示用途
GS voxelization3D voxel occupancy把连续 GS primitive 转成可规划几何。
Opacity filteringfiltered voxel map清掉地面附近低置信漂浮点，减少 free space 被误占用。
Height projectionrobot navigable map, human walkable map按不同高度范围投影到 2D occupancy。robot map 用近地面到 robot height；human map 用近地面到 human height。
Inflationinflated occupied grids按 robot radius / human radius 加 safety margin。

5. Human Planner

Human planner 的目标不是复现真实人类意图，而是生成 interaction-rich scenarios。它在 human walkable map 上采样 human start / goal，使 human path 与 robot start-goal line 交叉或平行；先用 A* 生成 seed path，再用 spline-based optimization 平滑成 2D trajectory，最后交给 human animation module 生成 SMPL motion。

6. Robot Expert Planner

Expert planner 用 Hybrid A* 加 motion primitive library，适合 non-holonomic ground vehicle / unicycle model。动态人只有在出现在 camera FOV 且接近 robot trajectory 的安全范围时，才会被加入 robot navigable map。

模块细节
Human occupancy extraction过滤 covariance trace 超过 75th percentile 的高不确定 Gaussian ellipsoids，再用 voxel-grid downsampling 得到更干净的 human points。
Dynamic safety buffer把 human points 投影到 2D robot map，并按 constant velocity prediction over 2 s horizon 膨胀。
Motion primitives3 个 velocity scales: 1/3、2/3、1 倍最大速度；3 个 steering commands: left、straight、right。
Search costprimitive arc length、steering penalty、heuristic cost-to-go 的 weighted sum。
Reactive replanning到达 replanning period，或 dynamic obstacle 与当前 look-ahead trajectory segment 的距离低于 safety margin 时触发。

7. Dataset Generation

在动态 GS simulator 中跑足够多 trials，记录训练样本:

样本字段含义
RGB observation由 static GS scene + animated human GS 从 robot viewpoint 渲染。
expert actionHybrid A* expert 输出的 linear velocity v 和 angular velocity w。
relative goal机器人 local coordinate frame 中的 Delta x、Delta y。

实验设置中，三个环境 Outside / Lobby / Library 分别用约 6 分钟 monocular video 重建，每个环境 1,000 到 1,500 张图；使用 6 个 human GS avatars；每个环境生成 400 training episodes，约 80k 到 120k samples；validation 为每环境 50 episodes。

8. Policy Training

项目内容
输入连续 3 帧 RGB images、previous action (v_{t-1}, w_{t-1})、relative goal (Delta x_t, Delta y_t)。
视觉编码器10 个 convolutional layers with residual connections，单帧输出 20-D encoded vector。
融合向量3 个 encoded vectors + previous action + relative goal，论文称为 64-D concatenated vector。
控制头3 层 MLP，输出 action (v_t, w_t)。
训练目标imitation learning，用 mean squared error 监督 policy action 匹配 expert action。
优化设置Adam，learning rate 1e-4；RGB resolution 144 x 256；帧间隔 0.5 s。

实验读法: 结果说明了什么

Simulation

ReaDy-Go 与 Vid2Sim 在 static 场景接近，但在 dynamic 场景差距明显。模拟中 ReaDy-Go dynamic SR: Outside 78%、Lobby 78%、Library 80%；Vid2Sim 分别为 57%、70%、68%。

Real World

真实世界中 ReaDy-Go dynamic SR: Outside 90%、Lobby 70%、Library 80%；Vid2Sim 为 60%、40%、60%；ViNT 为 30%、20%、40%。这支持“photorealistic dynamic GS obstacles 比非 photorealistic mesh assets 更利于 dynamic sim-to-real”。

Unseen Environment

把三个环境数据合并训练后，在 unseen env zero-shot 部署，Static SR 70%、Dynamic SR 50%，ART 分别为 30.60 s 和 35.45 s。它有泛化信号，但还不是强 open-world generalization。

对我们最重要的结论: environment-specific real-to-sim 数据在小模型上可以压过大规模 general navigation model；但 unseen environment 性能仍明显下降，说明需要更强的 scene completion、semantic/world priors 和跨环境数据规模。

Assumptions 与 Failure Modes

类型假设 / 风险对我们平台的启发
Scene acquisition目标环境视频采集时基本静态，且 monocular reconstruction + ArUco scale 足够可靠。对 partially observed scenes，需要显式记录 coverage map 和 uncertainty，而不是默认 GS 完整。
GS geometry弱纹理、反光、透明、细长物体可能导致 spurious / missing Gaussians；opacity threshold 会引入 free/occupied 误判。需要把 occupancy 的置信度暴露给 planner 和 completion module。
Dynamic humans动态物体只建模为 human GS；运动由 2D trajectory + motion prior 生成，缺少真实物理交互。我们的 asset system 应扩展到 carts、doors、chairs、vehicles、pets 等多类型动态对象，并区分 visual realism 与 physical realism。
PredictionHuman safety buffer 依赖 2 s constant velocity prediction，复杂人群行为会失效。可替换为 learned trajectory predictor 或 world-model rollout。
Policy inputpolicy 需要 relative goal，由 wheel odometry 估计；真实部署会有 drift。平台评估应加入 localization / odometry noise。
Generalizationunseen env 只有 50% dynamic SR，说明环境外泛化有限。open-world 方向应把 ReaDy-Go 当作 data generator，而非最终 generalist policy。
Simulation interface论文刻意避免 mesh extraction 和 physics engine integration。这让视觉数据生成变轻，但限制了 contact-rich、object manipulation、物理参数估计。

对 Collaborative Real2Sim 平台的可移植模块

我会优先移植的 5 个接口

  1. GSScene.to_occupancy(): 从 GS primitive 生成 3D voxel occupancy，并保留 opacity / uncertainty，而不是只输出 hard grid。
  2. DynamicActorGS: 把人、车辆、可移动物体都抽象成可渲染 dynamic GS actor，接口包含 visual asset、collision proxy、motion prior。
  3. TrajectoryConditionedAnimator: 输入 2D / 3D trajectory，输出时序 pose / joints / GS transform。ReaDy-Go 的 HUGS + PriorMDM 是 human-specific 实例。
  4. PlannerSupervisor: 用 Hybrid A* / MPC / sampling-based planner 在 GS-derived occupancy 上生成 expert demonstrations。
  5. ScenarioFactory: 自动采样 start-goal、人/物路径、视角、遮挡、动态事件，输出统一数据格式 (obs, state, action, goal, metadata)。

与 open-world / partially observed scene completion 的连接

ReaDy-Go 不会补全 unseen geometry。我们的改造点可以放在 static GS scene reconstruction 之后、planning map extraction 之前: 增加一个 Scene Completion Layer，对未观测区域、低置信区域、遮挡背后结构生成 hypothesis，并把每个 hypothesis 的 uncertainty 传给 planner / simulator / evaluation。

一个可行的 world-model-style loop 是: real video 重建 partial GS，抽取 coverage / occupancy uncertainty；用 semantic priors 或 generative world model 补全候选 layout / object assets；在 simulator 中 rollout 多个 plausible worlds；用 planner 和 policy 在这些 worlds 上训练，最终用 real-world validation 反向筛选最可信的 scene hypotheses。

一句话结论

ReaDy-Go 给我们的不是“自动补全开放世界”的答案，而是一个很实用的底座: 用 GS 把真实场景变成可渲染、可规划、可批量采样的数据工厂。把它接到我们的平台时，应把 GS-to-occupancy、dynamic actor injection、planner-generated supervision 先落地，再在低覆盖和高不确定区域接入 scene completion / world model。
